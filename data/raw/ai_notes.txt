LangChain 是一个用于构建大语言模型应用的开发框架，支持模型调用、Prompt 管理、工具调用、RAG、Agent 等能力。

Agent 是一种能够根据用户输入自主决定下一步行动的系统。与固定流程的 Chain 不同，Agent 可以动态选择是否调用工具，以及调用哪个工具。

RAG 是 Retrieval-Augmented Generation 的缩写，指通过检索外部知识来增强大模型回答能力的方法。它通常包含文档切分、向量化、向量检索和基于检索结果生成回答几个步骤。

FAISS 是 Facebook AI Similarity Search 的缩写，是一个高效的向量检索库，适合构建本地相似度搜索系统。

Embedding 是将文本映射到向量空间的技术。语义相近的文本在向量空间中的距离通常更近，因此可以用于语义检索。

向量数据库的核心能力是存储向量并支持相似度检索。对于小型项目，可以直接使用 FAISS 作为本地向量索引。

Prompt Engineering 是通过设计输入提示来改善大模型输出效果的方法，包括角色设定、任务说明、格式约束和示例提供等方式。

Transformer 是现代大语言模型的核心架构，其关键机制是 self-attention，可以对输入序列中不同位置的信息进行加权建模。

Self-attention 允许模型在编码一个 token 时关注输入序列中的其他 token，从而建模长距离依赖关系。

多头注意力机制可以让模型从不同子空间并行地学习不同类型的关联关系，提高表示能力。

Agent 的一个关键价值在于工具使用。大模型本身适合语言理解和生成，但不一定擅长精确计算、数据库查询或实时信息获取，因此需要借助外部工具。

Memory 用于让对话系统在多轮交互中保留上下文。短期记忆通常保存当前会话中的历史消息，长期记忆则用于跨会话持久化用户信息。

在工程项目中，Tool 通常被设计为独立模块，以便 Agent 按需调用，同时便于后续扩展更多能力。

一个典型的 AI Agent 项目可以包含：大模型推理层、工具层、记忆层、知识检索层以及运行入口。

在求职场景中，一个适合写进简历的 Agent 项目应突出模块化设计、多工具协作、检索增强和可扩展性。

如果用户问题涉及知识库中的概念性内容，可以优先使用 RAG 工具检索相关背景信息，再由 Agent 组织回答。

如果用户问题涉及数学表达式或精确数值计算，应优先调用计算工具，而不是依赖大模型直接心算。

好的工程实践包括：配置与代码解耦、敏感信息放入环境变量、模块分层清晰、提供基础测试脚本、便于扩展和维护。

LangChain 中的 Tool 可以被 Agent 自动选择调用，因此 Tool 的输入输出设计应尽量清晰稳定。

一个简单的短期记忆实现可以基于 thread_id 保存当前会话消息，供后续轮次继续使用。
